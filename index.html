𐎃 Що я думаю про переклади на власному досвіді? 𐎃

Перекладати моди не так важко, як здається. Деякі люди просто одразу дивляться на щось важке і велике, через це формується страх перекладів, люди починають боятися елементарних речей. Сьогодні, я допоможу вам не боятися і навчу вас перекладати

𐎃 Як перекладати насправді (по кроках) 𐎃
  1. Спочатку розпакуйте ваш мод в якусь папку
  2. Переходьте по шляху resource_pack/texts/en_US.lang (перші два слова "en_US" можуть відрізнятися) ось список мов: en_US - англійська, uk_UA - українська, ru_RU - російська, de_DE - німецька...
  3. Тепер відкривайте файл в якомусь текстовому редакторі (бажано в редакторі для коду, такому як "Acode"), і перекладаєте всі рядки ВАЖЛИВО: ключі "id:item.name" та інші НЕ ПЕРЕКЛАДАТИ, перекладаємо тільки текст після "=". Далі після перекладу зберігаємо файл правильно. Якщо ви перекладали на українську мову - зберігаємо файл як uk_UA.lang
  4. Ще не забудьте перевірити файли предметів у behaviour_pack/items/файл_предмету.json, шукайте там блок "minecraft:display_name": {, там внизу має бути "value" : і якщо там пише ключ з lang файлу - предмет перекладено, а як ні - треба вставляти самому
  5. Будь ласка, на останок перевірте наявність файлу languages.json в resource_pack/texts, а якщо його немає - треба просто створити! Приклад: 
    {
  "uk_UA": "Українська (Україна)",
  "en_US": "English (United States)",
  "en_UK": "English (United Kingdom)",
  "ru_RU": "Русский (Россия)",
  "pl_PL": "Polski (Polska)"
}

Як створювати моди

𐎃 Як створювати моди (по кроках) 𐎃
  1. Раджу спочатку зробити легкий мод, тому почніть створювати папки і файли, я розповім що кожне робить. Структура папок і файлів: 
    resource_pack/
│
├── manifest.json    //паспорт моду
├── pack_icon.png    //іконка моду
│
├── textures/    //текстури
│   ├── item_texture.json    //підєднання
│   ├── items/
│
├── texts/    //місце для перекладів
│   ├── en_US.lang
│   ├── uk_UA.lang
│   └── languages.json

====================================

behavior_pack/
│
├── manifest.json    //паспорт моду
├── pack_icon.png    //іконка моду
│
├── items/    //предмети
│   └── ваша_назва.json    //файл предмета
│ 
├── recipes/    //крафти предметів
│   └── ваша_назва.json    //файл для крафту
|
├── scripts/    //скрипти
│   ├──events    //файли скриптів
│   └── main.js    //під'єднання скриптів
  2. Файл manifest.json - це "паспорт" моду, назва строго manifest.json. Щоб він не конфліктував з іншими модами треба унікальний UUID, його можна отримати на сайті uuidgenerator.net
    Приклад resource_pack маніфест: 
    {
  "format_version": 2,
  "header": {
    "name": "Ваша назва (АНГЛІЙСЬКОЮ)",
    "description": "Опис (АНГЛІЙСЬКОЮ)",
    "uuid": "UUID",
    "version": [1, 2, 3],
    "min_engine_version": [1, 21, 0]
  },
  "modules": [
    {
      "description": "Resource Pack Module",
      "type": "resources",
      "uuid": "UUID",
      "version": [1, 2, 3]
    }
  ]
}
    Приклад behavior_pack маніфест: 
    {
  "format_version": 2,
  "header": {
    "name": "Ваша назва (АНГЛІЙСЬКОЮ)",
    "description": "Опис (АНГЛІЙСЬКОЮ)",
    "uuid": "UUID",
    "version": [1, 2, 3],
    "min_engine_version": [1, 21, 0]
  },
  "modules": [
    {
      "description": "Behavior Pack Module",
      "type": "data",
      "uuid": "UUID",
      "version": [1, 2, 3]
    },
    {
      "description": "Script Module",
      "type": "script",
      "language": "javascript",
      "uuid": "UUID",
      "version": [1, 2, 3],
      "entry": "scripts/main.js"
    }
  ],
  "dependencies": [
    {
      "module_name": "@minecraft/server",
      "version": "2.6.0"
    },
    {
      "module_name": "@minecraft/server-ui",
      "version": "2.0.0"
    }
  ]
}
    "format_version" - версія формату.
    "header", "name", "description" i "version" - основна інформація про мод.
    "uuid" - унікальний ключ.

    Все, що зв'язано зі скриптами та "dependencies" - поки не чіпаємо !ВАЖЛИВО! на місця UUID треба вписати ключ з сайту uuidgenerator.net

  3. Тепер поговоримо про pack_icon.png. pack_icon.png - це іконка моду, яка буде відображатися в Minecraft, назва строго pack_icon.png.
    Наприклад, може бути така іконка:
  4. Про textures і texts говорити не будемо, вони максимально легкі.

    Можу розповісти про файл item_texture.json.

    item_texture.json - це файл для під'єднання текстур. Назва строго item_texture.json.


    Вміст: 
    {
  "resource_pack_name": "M5566",
  "texture_name": "atlas.items",
  "texture_data": {
    "my_sword": {
      "textures": "textures/items/my_sword"
    }
  }
}
    "resource_pack_name" - назва ресурс паку (неважливо).

    "texture_name" - для чого файл, "atlas.items" - для предметів. Разом виходить: файл текстур для предметів

    "texture_data" - місце для текстур

    "my_sword" - назва текстури, яка буде використовуватись в інших файлах

    "textures" - повідомляє, що далі буде шлях до текстури, "textures/items/my_sword" - шлях до текстури.

    ПРОЧИТАЙТЕ! якщо текстура за шляхом textures/items/ називається my_sword.png, то в текстурному файлі item_texture.json треба писати textures/items/my_sword БЕЗ .png!

  5. Тепер поговоримо про items і файл предмета в behavior_pack.

    items - це папка в якій місцяться всі предмети, які мають з'явитися в Minecraft


    Наприклад, щоб зробити меч треба такий файл: 
    {
  "format_version": "1.21.10",
  "minecraft:item": {
    "description": {
      "identifier": "mymod:my_sword",
      "menu_category": {
        "category": "equipment"
      }
    },
    "components": {
      "minecraft:display_name": {
        "value": "назва (АНГЛІЙСЬКОЮ)"
      },
      "minecraft:icon": "my_sword",
      "minecraft:damage": 8,
      "minecraft:durability": {
        "max_durability": 782
      },
      "minecraft:max_stack_size": 1,
      "minecraft:hand_equipped": true,
      "minecraft:creative_category": {
        "parent": "equipment"
      },
      "minecraft:tags": {
        "tags": [
          "minecraft:is_sword"
        ]
      }
    }
  }
}
    !ВАЖЛИВО! числа і bolean (true, false), пишемо без лапок, коми ставимо після кожного блоку, крім останього.

    "format_version" - версія формату (НЕ ЗМІНЮВАТИ).

    "minecraft:item" - говорить, що це предмет.

    "identifier" - ID предмета, який показується при команді /give. Строго англійською і через двокрапку.слово:назва_предмета, відступи заборонені.

    "category" - категорія, в якій з'явиться предмет.

    "components" - всі дані про предмет, наприклад назва, іконка, шкода і довговічність.

    "minecraft:display_name" - назва предмету. В самому файлі можна писати тільки англійською, а через ланг якою хочете.

    "minecraft:icon" - наша іконка з файлу item_texture.json.

    "minecraft:damage" - шкода.

    "minecraft:durability" - довговічність.

    "minecraft:max_stack_size" - максимальна кількість в 1 стаку.

    "minecraft:hand_equipped" - тримати як інстумент. true - так, false - ні.

    "minecraft:creative_category" - підтвердження категорії.

    "minecraft:tags" - теги, для схожої поведінки.

  6. Рецепти... в Minecraft їх багато і ми додамо свій! Надіюсь, ви створили всі потрібні файли і папки.

    Приклад крафту в файлі recipes/:

    {
  "format_version": "1.20.10",
  "minecraft:recipe_shaped": {
    "description": {
      "identifier": "mymod:my_sword_recipe"
    },
    "tags": ["crafting_table"],
    "pattern": [
      " Y ",
      " Y ",
      " Y "
    ],
    "key": {
      "Y": {
        "item": "minecraft:stick"
      }
    },
    "result": {
      "item": "mymod:my_sword",
      "count": 1
    }
  }
}
    Тут є те, що ми з вами вже вчили, тому повторювати не буду.

    "minecraft:recipe_shaped" - каже, що крафт має форму.

    "pattern" - порядок предметів.

    "key" - "ключі" крафту.

    "Y" - назва "ключа".

    "item" - предмет, який означає "ключ", "minecraft:stick" - ID предмета ("minecraft:stick" - це палка)

    "result" - що і скільки скрафтиться, "item" - що отримаємо, "mymod:my_sword" - який предмет отримаємо, "count" - скільки штук отримаємо (в нашому випадку 1). Разом це все виходить: якщо викласти палки minecraft:stick вертикально, то ми отримаємо 1 меч "mymod:my_sword"
